# LabRemote 用户使用手册

## 1. 安装与首次启动

- Windows amd64：运行 `LabRemote-amd64-installer.exe`，或直接启动 `LabRemote-windows-amd64.exe`。安装包会在缺少 Windows WebView2 Runtime 时自动安装；
- Linux amd64：解压 `LabRemote-linux-amd64.tar.gz` 后运行 `LabRemote`。系统需安装 GTK3、WebKitGTK 4.1，并启动兼容 Freedesktop Secret Service 的系统钥匙串；
- macOS：解压 `LabRemote-macos-universal.zip` 并运行 `LabRemote.app`，同时支持 Apple Silicon 与 Intel。当前产物未签名或公证，首次启动可能需要在“隐私与安全性”中确认。

下载后请使用 Release 中的 `SHA256SUMS.txt` 核验文件完整性。

首次启动时 MCP 默认为关闭。LabRemote 不会自动授权任何服务器给 MCP。

## 2. 新建连接

在左侧“连接配置”标题区点击“新建”，填写：

### 基本信息

1. 连接名称；
2. 连接方式：
   - “隔离隧道 + SSH”：SSH 服务器只能通过实验室 SoftEther 网络访问时使用；
   - “仅 SSH（直接连接）”：SSH 服务器可由当前电脑直接访问时使用，不建立 VPN 或 SoftEther 隧道。

### 隔离隧道（仅“隔离隧道 + SSH”）

1. SoftEther 服务器名称或地址；
2. 端口，默认 `992`；
3. Virtual Hub，可留空让程序在服务器只公开一个 Hub 时自动发现；
4. 传输类型。当前发行版提供“SoftEther 原生协议（进程内隔离）”；
5. 用户名；
6. 密码。

### SSH

1. 服务器地址；
2. 端口，默认 22；
3. 用户名；
4. 认证方式：
   - “密码”：填写 SSH 密码；
   - “私钥文件”：点击“浏览…”选择 OpenSSH 或 PEM 私钥；加密私钥还需填写私钥口令，未加密私钥留空即可。

保存后，普通 JSON 配置只包含非敏感字段、连接方式、认证方式、服务器证书指纹和凭据引用。隧道密码、SSH 密码、私钥文件路径和私钥口令保存在当前用户的系统安全凭据库中：Windows Credential Manager、macOS Keychain 或 Linux Secret Service；私钥文件本身仍位于用户选择的原路径，不会被复制进 Profile JSON。编辑连接时凭据字段保持空白，表示保留原凭据。需要删除凭据时使用对应的“清除已保存…”按钮。

隔离模式保存前可以分别点击“测试隔离隧道”和“测试 SSH 服务器”：“测试隔离隧道”会验证 SoftEther 认证、DHCP 和用户态网络初始化，“测试 SSH 服务器”会继续验证 SSH 端口、主机指纹和所选认证方式。仅 SSH 模式只显示“测试 SSH 服务器”，直接验证当前电脑到 SSH 服务的端口、指纹和密码或私钥认证。测试使用临时配置与内存凭据，不会提前保存表单，也不会修改正式证书或 SSH 主机指纹信任记录。

LabRemote 不使用 Windows Split Tunneling。SoftEther、DHCP、ARP 和 TCP/IP 均在应用进程内运行。SSH 连接只能访问配置的 SSH 主机和端口；只有用户明确使用“网页访问”时，临时网页代理才会访问本次填写的 HTTP/HTTPS 目标。Windows VPN 状态、网卡和路由表不会改变。

## 3. 连接和终端

双击左侧连接，或选中后点击左侧列表下方的“连接”。隔离模式依次执行：

1. 验证并建立 SoftEther TLS 会话；
2. 在进程内获取 DHCP 地址并启动用户态 TCP/IP 栈；
3. 只为配置的 SSH 目标创建用户态 TCP 连接；
4. 验证 SSH 主机指纹并认证；
5. 打开 xterm-256color PTY。

仅 SSH 模式跳过前三个隧道步骤，使用操作系统网络直接连接配置的 SSH 主机与端口，然后执行 SSH 主机指纹验证、密码或私钥认证并打开 PTY。切换连接方式、SSH 认证方式、服务器端点或更新凭据并保存时，LabRemote 会关闭该配置原有的活动连接，避免继续复用旧传输路径。

第一次连接时会先展示 SoftEther 服务器证书的 SHA-256 指纹，再展示 SSH 主机密钥指纹。请分别与管理员提供的信息核对后确认。任一已固定指纹发生变化时，LabRemote 都会阻断连接。

终端快捷键：

- `Ctrl+Shift+C`：复制选择内容；
- `Ctrl+Shift+V`：粘贴；
- `Ctrl+F`：搜索终端输出；
- 右键：有选区时复制，无选区时粘贴。

多个标签可以共享同一进程内隧道。普通断开会在存在活动会话时拒绝；强制断开会先关闭该配置的全部会话。

左侧“连接配置”标题区集中提供新建、编辑和删除操作；选中连接后，编辑和删除按钮才可用。右键单个连接也可以连接、断开、打开文件传输、编辑或删除；右键操作会先选中对应连接，避免误操作其他配置。

## 4. 网页访问内网资源

选中目标连接后点击顶部“网页访问”，或右键连接并选择“网页访问…”，输入内网 HTTP/HTTPS 地址，例如：

```text
192.168.1.2:2512
http://192.168.1.2:2512/dashboard
http://127.0.0.1:1294
https://intranet.example.local
```

省略协议时默认使用 HTTP。LabRemote 会先按配置建立“隔离隧道 + SSH”或“仅 SSH”连接，再在随机本机端口启动仅监听 `127.0.0.1` 的临时反向代理，并通过系统默认浏览器打开带随机会话令牌的入口。浏览器资源请求、重定向、Cookie 和 WebSocket 连接会通过 SSH `direct-tcpip` 跳转到目标；目标域名由远端 SSH 主机解析。因此 `http://127.0.0.1:1294` 表示 SSH 服务器自身的 1294 端口，而不是运行 LabRemote 的本机端口，该服务无需监听公网地址或开放公网防火墙。

浏览器地址栏显示的是 `127.0.0.1` 临时地址，这是隔离代理入口，不是资源绕过隧道。未持有随机会话令牌的本机请求会被拒绝；代理授权 Cookie 不会转发给远端服务器。HTTPS 目标仍会验证远端证书，不会自动忽略无效或自签名证书。

网页访问代理会保持到用户右键选择“关闭网页访问”、删除连接、强制断开或退出 LabRemote。普通断开会在存在网页访问代理时提示确认，强制断开会同时关闭代理，已打开的浏览器页面随后不可再访问。若远端 SSH 服务禁用了 TCP 转发（OpenSSH 通常由 `AllowTcpForwarding` 控制），或 SSH 主机本身无法到达目标，界面会返回“SSH 跳转服务器无法访问浏览目标”。

如果网络波动导致 SSH 或隔离隧道意外断开，刷新已打开的网页时 LabRemote 会自动恢复可选隔离隧道和 SSH，并重新尝试本次网页连接；不需要重新创建本机代理地址。目标端口明确拒绝连接时不会重置健康的 SSH 会话。用户主动执行“强制断开”或“关闭网页访问”仍会关闭代理，不会在后台违背用户操作自动重连。

证书/主机指纹确认、强制断开、删除连接、清除凭据、取消传输等操作均使用 LabRemote 应用内确认弹窗。弹窗标题固定为 LabRemote，不再显示 WebView 来源的 `localhost` 浏览器式提示。

## 5. 上传和下载文件

选中左侧连接后点击顶部“文件传输”，并切换到“上传”页签。上传窗口支持：

1. 将文件或文件夹直接拖入窗口，或使用“选择文件”“选择文件夹”；
2. 混合添加多个文件和文件夹并递归保留目录结构；
3. 有当前连接的活动终端时，远端目标目录默认使用该终端实时所在目录；否则默认使用 `~`（SSH 用户主目录）；
4. 默认启用断点续传。取消或网络中断后，重新选择相同来源与目标即可复用安全分片；
5. 按需勾选“覆盖远端同名文件”。默认不覆盖，遇到同名文件会在传输前停止并提示；
6. 上传期间查看文件数、目录数、总字节、已续传字节、并发文件数和当前项目，也可以取消任务。

文件夹本身、层级结构和空目录都会保留。为避免循环或上传选择范围外的数据，符号链接不会被跟随；请改为选择其实际文件或文件夹。

点击顶部“文件传输”并切换到“下载”页签，可进入远端目录浏览器。双击文件夹进入目录，勾选一个或多个文件、文件夹后选择本地保存目录。下载会递归保留文件夹结构和空目录；远端符号链接不会被跟随。默认不覆盖本地同名项目，确认目标后可手动启用覆盖。

上传和下载均使用已认证 SSH 连接中的 SFTP 子通道。隔离模式继续经过 LabRemote 的进程内隧道；仅 SSH 模式直接连接 SSH，不创建 VPN、系统网卡或路由。一个任务最多同时处理 3 个文件，单个大文件使用多个并发 SFTP 请求。每个文件先写入目标同目录的确定性分片，成功后再原子提交；取消或可恢复错误会把分片截断到安全位置并保留。来源文件变化时会使用新分片，避免错误拼接。强制断开会同时取消该连接的上传和下载任务。

## 6. MCP

在左下角设置端口并开启 MCP。服务地址固定为：

```text
http://127.0.0.1:<端口>/mcp
```

开启全局 MCP 不会授权任何连接。需要编辑连接并明确勾选“允许 MCP 看到此配置”，再按需授予命令、交互会话、文件上传或断开权限。文件上传是独立权限，旧连接默认不会自动获得。

“复制 MCP 配置”会把地址和当前 Bearer Token 写入剪贴板，但不会写入应用日志。MCP 开启后还可以点击“导出 AI 终端操作手册”，保存一份 UTF-8 Markdown；文档包含当前 MCP 配置、已授权连接和权限、13 个工具的参数、非交互/交互终端与异步上传工作流、Base64/cursor 规则、错误处理与安全确认规范，可以直接提供给支持 MCP 的 AI 客户端。

导出的 Markdown 包含当前 Bearer Token，等同于本机 MCP 访问凭据。不要提交到 Git、工单或公开聊天；只应交给同一台电脑上受信任的 AI 客户端。怀疑泄露时点击“重新生成令牌”，旧令牌会立即失效。关闭 MCP 会停止新请求、取消 MCP 自己创建的上传并关闭 MCP 交互会话，不会关闭图形界面的 SSH 标签或传输任务。

首次通过 MCP 使用某个连接前，应先在图形界面连接一次：隔离模式确认 SoftEther 服务器证书与 SSH 主机指纹；仅 SSH 模式只需确认 SSH 主机指纹。MCP 上传要求 LabRemote 所在电脑上的绝对路径和明确的远端目标目录，默认不覆盖同名目标；任务通过 `job_id` 异步查询，支持并发和安全分片断点续传。完整的客户端配置、Profile 权限、13 个工具参数、安全边界和排障说明见 [MCP 使用、配置与安全手册](MCP配置与安全.md)。

## 7. 配置与日志位置

```text
%APPDATA%\LabRemote\profiles.json
%APPDATA%\LabRemote\settings.json
%APPDATA%\LabRemote\known_hosts
%LOCALAPPDATA%\LabRemote\logs\app-YYYY-MM-DD.jsonl
%LOCALAPPDATA%\LabRemote\logs\mcp-audit-YYYY-MM-DD.jsonl
```

终端输出、上传/下载进度和本地文件路径默认不持久化。卸载程序默认保留用户配置和 Credential Manager 中的秘密，避免误删；需要彻底清理时先在 LabRemote 内删除连接并选择同时删除凭据。
